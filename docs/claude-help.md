📚 ИНСТРУКЦИЯ: Новые команды, агенты и скиллы
После обновления конфигурации Claude Code в Jackdaw EDW теперь доступны следующие новые возможности:

🎯 Использование команд
/spec — Безопасное планирование задач
Сценарий: Планирование сложных архитектурных или многомаговых изменений без автоматического применения (избегает Plan Mode bug)

Использование:


/spec my-task-name
Эффект: Ввод в режим безопасного планирования. Claude НЕ будет выполнять изменения до вашего подтверждения с командой "proceed" (избегает авто-accept баг Plan Mode).

Дальнейшие шаги:

Выбирается фаза планирования (Explore → Design)
Claude пишет план в docs/plans/spec-my-task-name.md
Вы проверяете план и предлагаете правки
Вы вводите proceed для начала выполнения
Только после этого Claude начинает реализацию
Когда использовать:

Изменения схемы БД (ALTER TABLE, DROP TABLE, новые колонки)
Многомаговые задачи (10+ файлов одновременно)
Критические архитектурные решения
Когда НЕ использовать: простые правки кода, изменения в существующие задачи

/compact-edw — Компакция с сохранением EDW-состояния
Сценарий: Сохранение критеческого состояния перед компакцией контекста

Использование:


/compact-edw
Эффект: Выполняется компакция с автоматическим созданием snapshot в docs/plans/session-snapshot.md:

Сохраняемые данные:

Текущая задача — одно предложение о том, что делали
Модифицированные файлы — полный список путей всех изменённых файлов за сессию
DB-состояние — были ли изменения схемы БД, какие таблицы затронуты, обновлён ли schema.sql (yes/no)
SCD2-состояние — подтверждён ли хеш-формула, какие sync_status значения задействованы
Pending решения — нерешённые вопросы, TODOs, несвязанные FK/SCD2 решения
Блокеры — что не работало или было нечётко
Следующий шаг — ТОЧНОЕ действие после сброса контекста
Почему это важно:

При автокомпакции (когда CLAUDE достигает 70-90% контекста) теряется цель задачи, список файлов и состояние БД
Snapshot восстанавливает критический контекст после компакции
🤖 Использование агентов
/spec — Агент для планирования задач

/spec task-name
Описание: Специальный агент для создания планов реализации. Работает в два этапа:

Phase 1 (Explore) — только чтение, без изменений файлов
Phase 2 (Design) — написание плана в docs/plans/spec-task-name.md
Требует ваше подтверждение с proceed перед началом реализации
Использовать, когда: Нужен детальный план для многаговой задачи

/plan — Агент для выполнения планов

/plan task-name
Описание: Агент для выполнения планов, созданных через /spec.

Использовать, когда: Нужно выполнить план по шагам

/deploy — Агент для деплоя

/deploy flow-name
Описание: Агент для деплоя Prefect flows после одобрения кода.

Использовать, когда: Новый или изменённый flow готов к публикации

/test — Агент для тестирования

/test flow-name
Описание: QA-специалист для тестирования ETL flows.

Использовать, когда: После написания кода нужно протестировать

📋 Как работает сессия
Ленивая загрузка MCP инструментов (включено)

"ENABLE_TOOL_SEARCH": "auto:5"
Теперь MCP инструменты (pgedge, context7, etc.) загружаются только при использовании, а не при каждом старте сессии. Экономит ~50-80k контекстных токенов.

Компакция на 70% (новый триггер)

"CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "80"
Теперь контекст автоматически компактируется при достижении 70-80%, сохраняя EDW-состояние в snapshot.

Защита от flooding pgedge

"MAX_MCP_OUTPUT_TOKENS": "30000"
Ограничивает размер ответов от pgedge для защиты от переполнения контекста.

Sonnet для subagents

"CLAUDE_CODE_SUBAGENT_MODEL": "claude-sonnet-4-20250514"
Все агентные подзадачи теперь используют более дешёвую модель Sonnet вместо Opus.
